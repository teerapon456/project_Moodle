# สรุป: หน้าเรียน, UI ทั้งระบบ, และ HomePage แบบ CMU MOOC

## 1. หน้าเรียน (ที่ปรับใหม่แล้ว)

การนำทางในการเรียนที่ธีม Academi ปรับแล้วมีดังนี้:

| ส่วน | รายละเอียด |
|-----|------------|
| **Activity navigation** | แถบปุ่ม "ก่อนหน้า / ถัดไป" ใต้เนื้อหากิจกรรม — มีเส้นคั่นโทนมารูน ชัดเจน ตัวลิงก์สีมารูน |
| **Tertiary navigation** | เมนูเลือก Section/Topic (dropdown) — สีมารูน และ hover ให้โทนเข้มขึ้น |
| **Sticky footer** | แถบปุ่มล่าง (Continue, Previous section ฯลฯ) — เส้นคั่นมารูน ปุ่มหลักเป็นพื้นหลังมารูน |
| **Course index (drawer ซ้าย)** | รายการหลักสูตร/กิจกรรม — รายการที่อยู่ปัจจุบัน (pageitem) และ hover ใช้สีมารูน |
| **ปุ่มเปิด drawer** | ปุ่มเปิด Course index / Blocks — พื้นหลังมารูน |
| **Breadcrumb, ลิงก์กิจกรรม, Card header** | ใช้โทนมารูนใน content area อยู่แล้ว |

ไม่มีแผนปรับหน้าเรียนเพิ่มในรอบนี้ นอกจากต้องการเปลี่ยนสีหรือระยะห่างเพิ่มเติมสามารถบอกได้

## 2. UI การใช้งานระบบทั้งหมด — มีอะไรต้องปรับอีกไหม

โดยรวมธีมใช้โทนมารูนและ component หลักครบแล้ว สิ่งที่อาจพิจารณาเพิ่ม (ถ้าต้องการ) มีดังนี้:

- **ฟอร์ม (input/select):** ตอน focus อาจเพิ่มขอบหรือ shadow โทนมารูน (`outline` / `box-shadow` ด้วย `$primary`) ให้ชัดขึ้น
- **Modal / ปุ่มใน dialog:** ปุ่มหลักใน modal ใช้ `.btn-primary` อยู่แล้ว จะได้สีมารูนจาก preset
- **ข้อความแจ้งเตือน (alert):** success/warning/danger ยังเป็นสี Bootstrap มาตรฐาน ถ้าต้องการให้ success เป็นโทนมารูนสามารถ override ใน SCSS ได้
- **Dropdown ในเมนู:** รายการที่ hover/active ใช้สีตามธีมแล้ว

ถ้าไม่มีการรายงานจุดที่ "สีแปลก" หรือ "กดไม่รู้ว่ากดได้" เพิ่มเติม ถือว่า **ไม่จำเป็นต้องปรับอะไรเพิ่ม** สำหรับ UI ทั้งระบบในตอนนี้

## 3. HomePage — ทำแบบ CMU MOOC ได้ไหม / ใช้ได้แค่เท่าที่มี

### สิ่งที่ CMU MOOC (mooc.cmu.ac.th) มี

- Header + โลโก้ + สลับภาษา (TH/EN)
- หลายบล็อกบนหน้าเดียว: แต่ละบล็อกเป็น **หนึ่งหมวดหมู่** (เช่น CMUMOOC, CMUMOOC-I, CMU IDP) มีหัวข้อ + ปุ่ม "View All" + **การ์ดหลักสูตรในหมวดนั้น** (รหัส, ชื่อ, จำนวนผู้เรียน, FREE, หน่วยงาน)
- Pagination ต่อบล็อกหรือต่อหน้า
- Footer (ข้อมูลติดต่อ ฯลฯ)

### สิ่งที่ธีม Academi มีอยู่แล้ว (ใกล้เคียง CMU MOOC)

| ฟีเจอร์ | Academi |
|--------|---------|
| สไลด์โชว์ (Hero) | มี — ตั้งค่าจำนวนสไลด์และรูปได้ |
| แถบหมวดหมู่ | มี — แถบปุ่มลิงก์ไปหน้ารายการหลักสูตรในหมวด (เรียงชื่อ/จำนวนหลักสูตร, เลือกได้ว่าหมวดไหนบ้าง) |
| รายการหลักสูตรแบบการ์ด | มี — ตามรูปแบบ Moodle (coursebox) + ปรับ layout การ์ดใน frontpage ได้ |
| Pagination | มี — จำนวนหลักสูตรต่อหน้า (6, 9, 12, 18, 24) ตั้งค่าได้ |
| Footer บล็อก + โซเชียล | มี |
| หลักสูตรแนะนำ (จาก category / ล่าสุด) | มี (จาก theme settings) |

### สิ่งที่ทำได้แค่เท่าที่มี (ข้อจำกัด)

- **หลายบล็อกต่อหน้าแบบ CMU (หลายหมวด พร้อมการ์ดในแต่ละหมวด):**  
  Moodle หน้าแรกมาตรฐานแสดงได้แบบเดียวต่อครั้ง เช่น "รายการหลักสูตรทั้งหมด" หรือ "รายการหลักสูตรในหมวดที่เลือก" **ไม่มีการ์ดหลายหมวดบนหน้าเดียว** แบบ CMU โดยไม่เขียนโค้ดเพิ่ม  
  - ถ้าต้องการ **หลาย section แบบ CMU** (เช่น Section 1 = หมวด A + การ์ด, Section 2 = หมวด B + การ์ด) ต้องใช้อย่างใดอย่างหนึ่งต่อไปนี้:
    - **Block/Plugin:** เช่น block ที่แสดง "รายการหลักสูตรในหมวด X" แล้วใส่หลายบล็อกคนละหมวด หรือ plugin หน้าแรกแบบ custom
    - **แก้ใน theme:** ให้ frontpage ดึงหลายหมวดมาเรียงเป็นหลาย section และแต่ละ section แสดงการ์ดหลักสูตรในหมวดนั้น (ต้องเขียน PHP/HTML เพิ่มใน layout หรือ lib แล้วส่งข้อมูลเข้า template)

สรุป:

- **ทำแบบ CMU MOOC แบบ "มี Hero + แถบหมวด + รายการหลักสูตรการ์ด + pagination + footer"** → **ได้** และใกล้เคียงแล้วด้วยฟีเจอร์ที่มี
- **ทำแบบ CMU MOOC แบบ "หลายบล็อก (หลายหมวด) บนหน้าเดียว แต่ละบล็อกมีการ์ดหลักสูตรของหมวดนั้น"** → **ใช้ได้แค่เท่าที่มีไม่ได้** ต้องมี plugin/block หรือแก้ theme เพิ่ม (โค้ดค่อนข้างใหญ่)

ถ้าต้องการเดินทางไปทาง "หลายหมวดบนหน้าเดียว" สามารถวางแผนเป็น Phase 2 ได้ (เช่น เริ่มจาก block "Course list by category" แล้วใช้หลายบล็อกบน frontpage หรือเขียน section หลายหมวดใน theme).

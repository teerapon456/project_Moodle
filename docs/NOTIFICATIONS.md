## Dormitory — ข้อความแจ้งเตือนตามประเภทคำขอ

| request_type   | ตอนอนุมัติ (In-App + อีเมล) | ตอนปฏิเสธ |
|----------------|-----------------------------|-----------|
| `move_in`      | คำขอเข้าพักได้รับการอนุมัติ กรุณารับกุญแจ [วันที่] | คำขอเข้าพักถูกปฏิเสธ: [เหตุผล] |
| `move_out`     | คำขอย้ายออกได้รับการอนุมัติแล้ว | คำขอย้ายออกถูกปฏิเสธ: [เหตุผล] |
| `change_room`  | คำขอเปลี่ยนห้องได้รับการอนุมัติแล้ว [วันที่ดำเนินการ] | คำขอเปลี่ยนห้องถูกปฏิเสธ: [เหตุผล] |
| `add_relative` | คำขอเพิ่มญาติได้รับการอนุมัติแล้ว | คำขอเพิ่มญาติถูกปฏิเสธ: [เหตุผล] |
| `remove_relative` | คำขอนำญาติออกได้รับการอนุมัติแล้ว | คำขอนำญาติออกถูกปฏิเสธ: [เหตุผล] |

## ข้อควรระวัง / ข้อจำกัด

| หัวข้อ | รายละเอียด |
|--------|-------------|
| **In-App ต้องมี user ในระบบ** | ถ้า email ไม่มีในตาราง `users` หรือ `is_active = 0` จะไม่สร้างแจ้งเตือน In-App (ส่งอีเมลได้ถ้ามีเมธอดส่งอีเมล) |
| **แอดมินต้องตั้งค่า** | `system_settings` ของแต่ละโมดูลต้องมี `admin_emails` (คั่นด้วย comma) ถึงจะแจ้งแอดมินได้ |
| **Cron in_use** | ต้องรัน `cron_update_status.php` (เช่น ทุก 5 นาที) ถึงจะเปลี่ยนสถานะและส่งแจ้ง "ถึงเวลาเริ่มใช้" |
| **ลิงก์ในแจ้งเตือน** | ถ้าเรียกจาก Cron ใช้ path อย่างเดียว (ไม่มี base URL) ฝั่งแสดงแจ้งเตือนต้องต่อ base เอง |

## วิธีทดสอบการแจ้งเตือน (สั้นๆ)

1. **In-App** — ตรวจตาราง `notifications` ว่ามีแถวใหม่หลังทำเหตุการณ์ (user_id, type, title, message, link).
2. **อีเมล** — ตรวจจาก Email Log (ถ้ามี) หรือกล่องจดหมายทดสอบ.
3. **Car Booking Cron** — รัน `php Modules/CarBooking/cron_update_status.php` แล้วตรวจสถานะ `in_use` และแจ้งเตือนผู้ขอ.

---

*อัปเดตตามการแก้ไข CarBooking (cron in_use, report_return) และ Dormitory (ข้อความ approve/reject ครบทุก request_type)*
